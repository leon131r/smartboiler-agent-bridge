# Умный Котёл v4.2 (Модульный)

Рефакторинг проекта с разделением на модули для удобства поддержки.

## 📁 Структура проекта

```
SmartBoiler_JSON_v4/
├── SmartBoiler_JSON_v4.ino   # Только setup() и loop()
├── config.h                  # Настройки WiFi, пины, константы ⚠️ секреты!
├── sensors.h / sensors.cpp   # Чтение NTC, проверка безопасности
├── control.h / control.cpp   # Брезенхем, авто-мощность, реле
├── telegram.h / telegram.cpp # Telegram бот, очередь, пользователи
├── webserver.h / webserver.cpp # Веб-сервер, API endpoints
├── storage.h / storage.cpp   # Preferences (настройки, пользователи)
└── data/                     # LittleFS файлы
    ├── index.html            # Веб-интерфейс
    ├── style.css             # Стили
    └── app.js                # JavaScript приложение
```

## ⚠️ ВАЖНО: Секреты

Файл `config.h` содержит WiFi пароль и Telegram токен.
**Не коммитьте его в публичные репозитории!**

Перед компиляцией измените в `config.h`:
```c
// Основная сеть
#define WIFI_SSID_1 "MyHomeWiFi"
#define WIFI_PASS_1 "password123"

// Резервная сеть (или оставьте "" если не нужна)
#define WIFI_SSID_2 "MyPhoneHotspot"
#define WIFI_PASS_2 "hotspot456"

#define BOT_TOKEN "YOUR_BOT_TOKEN"
```

### Как работает WiFi-переключение
1. При запуске подключается к **сети #1**, если недоступна — к **сети #2**
2. При потере связи сначала пробуется переподключение к текущей сети
3. Если не удалось — автоматическое переключение на резервную
4. В автономном режиме проверка каждые 10 минут
5. Каждая сеть пробуется **2 раза** с таймаутом **8 секунд**

## 🚀 Установка

### 1. Arduino IDE
- Установите платформу ESP32
- Установите библиотеки:
  - `ArduinoJson` (6.x)
  - `UniversalTelegramBot`
  - `WebServer` (встроена)
  - `WiFi` (встроена)
  - `Preferences` (встроена)
  - `LittleFS` (встроена)

### 2. Загрузка файлов в LittleFS
- Установите плагин ESP32FS
- Скопируйте содержимое `data/` в проект
- `Tools → ESP32 Sketch Data Upload`

### 3. Прошивка
- Подключите ESP32
- Выберите плату и порт
- Нажмите Upload

## 🌐 API Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Веб-интерфейс |
| GET | `/api/status` | Статус системы (JSON) |
| GET | `/api/history` | История температур |
| POST | `/api/control` | Команды управления |

### Команды управления

| Команда | Описание |
|---------|----------|
| `on` | Авто режим |
| `off` | Выключить |
| `manual` | Ручной режим |
| `tempXX` | Цель XX°C (temp40) |
| `powerXX` | Мощность XX% (power50) |
| `maxXX` | Макс. температура (max75) |
| `reset` | Сброс ошибок |

## 📡 Telegram команды

### Наблюдатель
- `/status` — статус системы
- `/temp` — температуры
- `/water` — уровень воды
- `/info` — информация

### Пользователь
- `/on` — авто режим
- `/off` — выключить
- `/manual` — ручной режим
- `/tempXX` — цель XX°C
- `/powerXX` — мощность XX%
- `/maxXX` — макс. температура

### Владелец
- `/users` — список пользователей
- `/adduser [уровень] [chat_id]`
- `/removeuser [chat_id]`
- `/reset` — сброс ошибок
- `/restart` — перезагрузка

### Регистрация
```
/register kotel2024
```
Первый пользователь становится Владельцем.

## 🔧 Технические детали

### Алгоритм Брезенхема
- Частота: 10 Гц (100 мс)
- Плавное управление ШИМ без мерцания
- Защита от дребезга реле: 100 мс

### Датчики NTC
- Двойное чтение с усреднением
- Адаптивные коэффициенты (холодный/тёплый диапазон)
- Проверка на обрыв/КЗ

### Безопасность
- Критическая температура: 85°C
- Защита по подаче (верхняя точка)
- Контроль уровня воды (геркон)
- Автоматическое отключение ручного режима при перегреве

### Автономный режим
- При потере WiFi — полная работа по расписанию
- Периодическая проверка сети (каждые 10 мин)
- Автоперезагрузка при восстановлении сети

## 🐛 Отладка

- Ошибки инициализации
- Статус WiFi
- События Telegram

### Браузер (F12)
- Network — API запросы
