# AGENTS.md — Git Mailbox Protocol

> Файл для **обоих агентов** (Qwen на хосте + OpenClaw в Docker).
> Это **единый канал связи** через Git-репозиторий.

## 📋 Правило #1: Всегда делай `git pull` перед работой

```bash
git pull --rebase origin main
# ... делаешь изменения ...
git add .
git commit -m "[префикс] описание"
git push origin main
```

## 📬 Формат сообщений (префиксы коммитов)

| Префикс  | Кто пишет | Значение |
|----------|-----------|----------|
| `[task]` | Агент A  | Задание для другого агента |
| `[done]` | Агент B  | Задание выполнено |
| `[info]` | Любой    | Информация/контекст |
| `[ask]`  | Любой    | Вопрос другому агенту |
| `[fix]`  | Любой    | Баг-фикс |
| `[feat]` | Любой    | Новая функция |
| `[sync]` | Любой    | Синхронизация состояния |

## 📝 Примеры

### Агент A ставит задачу:
```bash
git commit -m "[task] Рефакторинг webserver.cpp — вынести CORS в конфиг"
```

### Агент B выполняет:
```bash
git commit -m "[done] webserver.cpp: CORS вынесен в config.h, протестировано"
```

### Агент A подтверждает:
```bash
git commit -m "[sync] Принято, webserver.cpp выглядит ок"
```

## 🔄 Файлы-мессенджеры (для сложной коммуникации)

Для длинных сообщений создавай файлы в `agent-mail/`:

```
agent-mail/
├── FROM_QWEN.md      # Сообщение от Qwen → OpenClaw
├── FROM_OPENCLAW.md  # Сообщение от OpenClaw → Qwen
└── ARCHIVE/          # Прочитанные сообщения
```

### Протокол работы с agent-mail/:

1. **Написать**: создай `agent-mail/FROM_ME.md`, запиши сообщение
2. **Отправить**: `git add agent-mail/FROM_ME.md && git commit -m "[info] new mail" && git push`
3. **Прочитать**: `git pull`, проверь `agent-mail/FROM_OTHER.md`
4. **Ответить**: перемести прочитанное в `ARCHIVE/`, создай свой `FROM_ME.md`

## ⚠️ Важно

- **НЕ КОММИТЬ** `config.h` (содержит токены)
- **НЕ РЕШАЙ** чужие `[task]` — только тот кому адресовано
- **ВСЕГДА** `git pull --rebase` перед `git push`
- **НЕ ПЕРЕПИСЫВАЙ** историю (`push --force` запрещён)
